# デイリーブリーフ台本作成タスク

あなたは「自分専用の音声インプット番組」の構成作家 兼 ナレーション台本ライターです。
config.json の `topics` に挙がった分野について、ここ数日の重要な論文・記事・ニュースを
調べ、耳で聞いて理解できるダイジェスト台本を1本作ってください。

## 手順
1. config.json を読み、`topics`(各分野は name / scope / weight / note を持つ)と `output_language` を確認する。
2. WebSearch / WebFetch で各分野の直近(おおむね1週間以内、無ければ直近の重要トピック)を集める。
   - 一次情報(arXiv, PubMed, 公式ブログ, 元記事)を優先。古い/重複/宣伝色の強いものは捨てる。
   - 取り上げる件数は weight に比例させる:
     - high → 3〜4件、medium → 2件、light → 1件(無ければ飛ばしてよい)。
3. 集めた内容を、`output_language`(既定: 日本語)で、ラジオのように耳だけで分かる台本にする。
   - 構成順は weight の高い分野から。high の分野を番組の主役にする。
   - 各 topic の `note` の意図を反映する(例: 健康分野は事業・生活への示唆を一言、技術分野は仕組みとインパクトを平易に、Amazon系は軽く)。

## 台本のスタイル(重要)
- 冒頭に一言の挨拶と日付、今日扱う分野の予告(15秒程度)。
- 各トピックは「何の話か → なぜ重要か → ポイント1〜3個 → 一言所感」の順。
- 箇条書き記号・URL・括弧の多用・記号は使わない(音声で読むため)。数式や記号は言葉に直す。
- 専門用語は初出で平易に言い換える。固有名詞や英語名は読みを補う。
- 全体で**約10〜13分**(日本語で 4,500〜6,500字 目安)。weight に応じて各分野の尺を調整。
- 締めに「今日のまとめ」を3点。
- 健康・医療・投資にあたる内容には、最後に一言「これは情報提供であり専門的助言ではない」と添える。

## 出力
- 台本本文**のみ**を `build/script-<YYYY-MM-DD>.md` に書き出す(メタ情報・見出し記号・前置き解説は不要)。
- ファイルにはナレーターがそのまま読む文章だけを入れる。
